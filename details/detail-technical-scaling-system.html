⚖️ Scaling System

Hybrid Scaling Approach

Balanced computational efficiency with economic realism

  • Agent counts scaled: Firms and households divided by SIMULATION_SCALE_RATIO (1:1000)
  • Economic indicators real: GDP, population, unemployment rates at real-world values
  • Prices & wages real: All monetary values maintained at real scale
  • Coefficients real: Technical coefficients and ratios at empirical scale

Scale Conversion Process

Automatic upscaling for meaningful economic analysis

  • Single source of truth: SIMULATION_SCALE_RATIO from DatabaseConfig.properties
  • Scale-at-point-of-use: Data stored at simulation scale, upscaled when needed
  • Clear naming conventions: Explicit distinction between sim-scale and real-scale values
  • Automatic upscaling: Micro-level behavior aggregates to macro-level outcomes

Implementation Example

Practical scaling in economic calculations

  • Agent scaling: 3.5M firms → 3,500 simulation firms (1:1000 ratio)
  • Upscaling calculation: realLaborStock = laborStock × SIMULATION_SCALE_RATIO
  • Economic consistency: Total labor cost = realLaborStock × avgWage
  • Validation: Simulation outputs match real-world economic patterns
